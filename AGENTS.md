# NubHub

## Purpose

This repository holds three independent Node.js projects that together run the
NubHub product (AI + expert-reviewed "nub theory" baby-gender prediction from a
12-week ultrasound scan, at nubhub.baby) and the marketing/content operation
around it:

- **`app/`** — the production Next.js web application (upload a scan, pay,
  get an AI + human-reviewed prediction; plus a ~680-article SEO content
  library).
- **`content-pipeline/`** — a standalone Node script that generates and posts
  daily AI video reels (TikTok/Instagram) via Higgs Field.
- **`SlideSmith/`** — an internal Vite + React + Express tool for generating
  and scheduling social slideshows/carousels.

These are **not** a monorepo: each has its own `package.json`, lockfile, and
`node_modules`. There is no shared root workspace config — `cd` into the
relevant project before running its scripts.

## For AI Agents

### Working In This Directory

- **Branch model is not the usual one.** Day-to-day development happens on
  `main`, where the Next.js app lives in `/app`. Production (nubhub.baby) is
  deployed by Vercel from the `master` branch, where the *same* app content
  is flattened to the **repo root** via a subtree-split, then force-pushed to
  `master`. Never hand-edit `master` directly — always work on `main` and
  publish through the subtree-split process.
- **Stripe is LIVE in production** (since 2026-06-25, dedicated NubHub
  account). Treat `app/lib/stripe.ts`, the checkout API routes, and webhook
  handlers as production-payment code — verify against Stripe test mode
  before touching anything in that path.
- Secrets (Stripe keys, etc.) are loaded via the PowerShell script pattern
  (`set-nubhub-stripe-live-env.ps1`), never pasted into chat or committed.
  Follow the same pattern for any new secret.

### Testing Requirements

No unified test runner across the repo — each subproject has its own
conventions. See the nested `AGENTS.md` in `app/`, `content-pipeline/`, and
`SlideSmith/` for specifics (e.g. `app/e2e-test.mjs`,
`content-pipeline`'s `dry_run` config flag).

### Common Patterns

- Three sibling Node projects, each independently installed/built/run.
- Secrets are never hardcoded — loaded from local env/config files that are
  gitignored (`content-pipeline/config.json`, `.mcp.json`, etc.).

## Dependencies

### Internal

`content-pipeline/` and `SlideSmith/` are operationally independent of
`app/` (no shared imports) but support the same product — they generate
marketing content that links back to nubhub.baby.

### External

Each subproject pins its own dependencies — see its `package.json` /
`AGENTS.md`. Notable cross-cutting third-party services: Stripe (payments),
Vercel (hosting), Neon (Postgres), Cloudflare R2 (storage), Higgs Field (AI
video generation), Resend (email), Telegram (review workflow bot).
